3 dagers kurs om Testautomatisering med Perl

2. til 4. april, i Linpro sine lokaler på Storo i Oslo. Kurset holdes på engelsk.

Perl har i mange år blitt brukt i kvalitetssikring (QA) til alt fra forberedelse av konfigurasjonsfiler før testing til analyse av logger etter testen. Utviklingen har ikke stått stille, og idag kan Perl brukes i alle deler av et automatisk testmiljø - inkludert det å simulere brukere ved å styre mus og manipulere applikasjoner under Windows, og fungere som en web-klient for å teste web-baserte applikasjoner.

Målgruppe

Programmerere og testere med kjennskap til Perl.

Emner

Oversikt over hvordan Perl kan brukes til å automatisere kvalitetssikring. Anvende Perl i ulike typer testing (som GUI, web, nettverk). Bruk av testrammeverket som kommer med Perl.

Kursformat

24 klasseromstimer (3 fulle dager). Omtrent 50% av tiden til øvelser.

Forkunnskaper

Seks måneders erfaring med programmering i Perl, eller introduksjonskurs i Perl

Innhold

Først vil vi lære om de grunnleggende testverktøyene og -rammeverkene som er tilgjengelig i Perl ved å se på unit-testing av Perlmoduler. Deretter utvider vi verktøysettet for å teste andre typer applikasjoner. Vi vil se på testing av Windows brukergrensesnitt, webapplikasjoner og nettverksutstyr

Stikkord

  • Introduksjon til testing: hvorfor, når, hvor, hvem og hvordan?
  • Perl sine verktøy for testing av Perlmoduler
  • Microsoft Windows GUI applikasjoner
  • Kommandolinjeapplikasjoner
  • Testing av nettverksutstyr med kommandolinjegrensesnitt
  • Web-applikasjoner enten med tekstnettleser eller med Internet Explorer
  • Databasetesting
  • X-Windows testing
  • Kort om XP - Extreme Programming
  • Bygge regresjonstestverktøy
